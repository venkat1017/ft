Lesson 07 of 10 - Measurement

Evaluation &
Benchmarks

⏱ 55 min📐 LM-Eval Harness🤖 LLM-as-Judge

A model that scores well on your training loss might be worse at the actual task. Evaluation is where most fine-tuning projects fail silently. We cover three complementary approaches: task-specific benchmarks, LLM-as-judge scoring, and human vibes testing - and when to use each.

The Evaluation Pyramid

Level 3: Human Evaluation (ground truth)
Have real users rate outputs. Expensive, slow, but most reliable. Run monthly or per major version. N=50 samples minimum.
Level 2: LLM-as-Judge (fast proxy)
Use GPT-4 or Claude to score your model's outputs. Correlates ~0.85 with human ratings. Run per epoch. N=200 samples. Use MT-Bench prompts.
Level 1: Benchmark Scores (fast, automated)
lm-evaluation-harness on standard benchmarks. Catches regressions instantly. Run per commit. Tells you if you broke something, not if you improved it.

llm_judge.py - score your model with GPT-4
LLM-AS-JUDGE
from openai import OpenAI

client = OpenAI()

JUDGE_PROMPT = """You are an expert evaluator. Rate the assistant response on:
1. Helpfulness (1-5): Does it answer the question fully?
2. Accuracy (1-5): Are facts correct?
3. Clarity (1-5): Is it well-organized and clear?

User question: {question}
Assistant response: {response}

Respond only in JSON: {{"helpfulness": N, "accuracy": N, "clarity": N, "reasoning": "..."}}"""

def judge_response(question, response):
    result = client.chat.completions.create(
        model="gpt-4o-mini",  # cheap but good enough for scoring
        messages=[{"role": "user", "content": JUDGE_PROMPT.format(
            question=question, response=response
        )}],
        response_format={"type": "json_object"},
    )
    import json
    return json.loads(result.choices[0].message.content)

What You Now Know

  • Three evaluation levels: benchmarks (fast), LLM-judge (proxy), human (ground truth)
  • lm-evaluation-harness: standard benchmarks to catch regressions
  • LLM-as-judge: GPT-4o-mini scoring correlates ~0.85 with human ratings
  • Always compare fine-tuned vs base model on the SAME benchmark
  • Task-specific evals beat generic benchmarks for measuring your use case
