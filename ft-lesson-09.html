DPO: Preference
Fine-Tuning

⏱ 70 min🎯 Alignment✅ Chosen vs Rejected

SFT teaches the model what to say. DPO (Direct Preference Optimization) teaches it which response is better when it could say multiple things. By training on (chosen, rejected) response pairs, DPO produces models that are more helpful, more honest, and refuse harmful requests more reliably - without needing a reward model.

Why DPO After SFT?

SFT trains the model to imitate examples. But good behavior isn't just about imitation - it's about preferences. A model trained only on SFT might give two equally fluent but very different quality responses. DPO directly optimizes for the preference: response A is better than response B. It's the step that turns a capable model into a reliable one.

DPO vs RLHF

RLHF: Train a reward model, then use PPO (complex RL algorithm) to optimize against it. 3 models, 3x memory, unstable training. DPO: Mathematically equivalent to RLHF but rearranges to a simple supervised loss. 1 model, same memory as SFT, stable. DPO has largely replaced RLHF for most use cases.

The DPO Loss Intuition

The DPO objective looks intimidating but the intuition is simple: increase the relative probability of the chosen response compared to the rejected one, while staying close to the SFT model (via an implicit KL penalty controlled by beta).

dpo_intuition.py - what DPO is actually optimizing
INTUITION
import torch

# DPO loss for one (prompt, chosen, rejected) triplet:
#
# loss = -log(sigmoid(beta * (log_ratio_chosen - log_ratio_rejected)))
#
# where:
#   log_ratio_chosen  = log P_model(chosen|prompt) - log P_ref(chosen|prompt)
#   log_ratio_rejected = log P_model(rejected|prompt) - log P_ref(rejected|prompt)
#   beta = regularization (higher = stay closer to reference model)
#
# Intuition:
#   - If model assigns much higher prob to chosen than ref → big positive reward
#   - If model assigns much higher prob to rejected than ref → big negative reward
#   - Beta controls how much we allow the model to deviate from the SFT baseline
#   - beta=0.1: allow large deviation (aggressive fine-tuning)
#   - beta=0.5: conservative (stay close to SFT model)

def dpo_loss(chosen_logps, rejected_logps, chosen_ref_logps, rejected_ref_logps, beta=0.1):
    chosen_ratios  = chosen_logps  - chosen_ref_logps   # how much better than ref?
    rejected_ratios = rejected_logps - rejected_ref_logps
    logits = beta * (chosen_ratios - rejected_ratios)
    loss   = -torch.nn.functional.logsigmoid(logits).mean()
    # reward_chosen  = beta * chosen_ratios.mean()  (track this - should rise)
    # reward_rejected = beta * rejected_ratios.mean() (track this - should fall)
    return loss

def generate_dpo_data(sft_model, prompts, n_responses=4):
    examples = []
    for prompt in prompts:
        responses = [generate(sft_model, prompt, temperature=0.9) for _ in range(n_responses)]
        scores    = [judge_response(prompt, r)["helpfulness"] for r in responses]
        best_idx  = scores.index(max(scores))
        worst_idx = scores.index(min(scores))
        examples.append({"prompt": prompt, "chosen": responses[best_idx], "rejected": responses[worst_idx]})
    return examples
run_dpo.py - DPO training with TRL
CORE CODE
from trl import DPOTrainer, DPOConfig

dpo_config = DPOConfig(
    beta=0.1,                    # regularization (0.1 = standard)
    max_length=1024,
    max_prompt_length=512,
    output_dir="./llama3-dpo",
    num_train_epochs=1,          # DPO: usually 1-3 epochs only
    per_device_train_batch_size=2,
    gradient_accumulation_steps=8,
    learning_rate=5e-7,          # MUCH lower than SFT lr (1e-6 to 5e-7)
    bf16=True,
    optim="paged_adamw_32bit",
)

trainer = DPOTrainer(

What You Now Know

  • DPO trains on (prompt, chosen, rejected) pairs - no reward model needed
  • Beta controls regularization: higher = stay closer to SFT baseline
  • DPO LR should be 10-100x lower than SFT LR (5e-7 is typical)
  • Self-play DPO: generate multiple responses, judge them, use best/worst as pairs
  • Monitor reward_chosen (should rise) and reward_rejected (should fall)
