Lesson 04 of 10 - Configuration

LoRA
Hyperparameters

⏱ 55 min⚙️ Rank, Alpha, Targets🎯 Where to Inject

LoRA has four critical hyperparameters: rank (r), alpha, dropout, and target modules. Choosing them correctly cuts training time in half and doubles model quality. Most tutorials cargo-cult r=16, alpha=16 without understanding why - this lesson builds the intuition to make deliberate choices.

Rank (r): The Capacity Knob

Rank controls how many parameters the adapter has. A rank-r adapter adds two matrices per target layer: (in_dim × r) and (r × out_dim). Higher rank = more expressive adapter = more parameters to train. But above a certain point, extra capacity doesn't help - it just overfits.

💡
Rule of thumb: Start with r=16. If val loss plateaus early (underfitting), try r=32. If val loss diverges from train loss (overfitting), try r=8 or add dropout. For instruction tuning on general tasks with 1K+ examples, r=16 is almost always right.

Alpha: The Scaling Factor

The LoRA output is scaled by alpha/r. This controls how much the adapter contributes relative to the frozen base. If alpha=r, the scale is 1.0. If alpha=2r, the adapter contributes twice as much. Why does this matter? Because it lets you change the effective learning rate of the adapter without changing your global LR.

alpha_intuition.py
INTUITION
# Forward pass with LoRA:
# output = base_output + (B @ A @ x) * (alpha / r)
#
# Common choices and their effective scales:
# alpha=r     → scale=1.0  (standard, most common)
# alpha=2*r   → scale=2.0  (stronger adaptation, riskier)
# alpha=r//2  → scale=0.5  (conservative, good for large datasets)
#
# Practical advice:
# - If you increase r, keep alpha=r (don't change relative scale)
# - If training is unstable: reduce alpha (or equivalently, reduce LR)
# - alpha does NOT need to be tuned independently from r in most cases
# - Setting alpha=r is equivalent to not having the scaling at all
#   (since scale=1.0) - it's essentially a passthrough

# Example from PEFT library:
from peft import LoraConfig

config = LoraConfig(
    r=16,        # rank
    lora_alpha=16,  # alpha = r → scale = 1.0
    # If you want stronger adaptation without increasing r:
    # lora_alpha=32  → scale = 2.0
)

Target Modules: Where to Inject LoRA

You can apply LoRA to any linear layer. But not all layers are equal. The attention Q,K,V,O projections are the highest-leverage targets - they control information routing and are where most task-specific learning happens. The FFN layers (gate_proj, up_proj, down_proj) add more capacity but also more parameters.

target_modules.py - what to target and why
CORE CONFIG
from peft import LoraConfig, get_peft_model

# ── Minimal: attention only (fewest params, fastest, often sufficient) ──
config_minimal = LoraConfig(
    r=16,
    lora_alpha=16,
    lora_dropout=0.05,
    bias="none",
    task_type="CAUSAL_LM",
    target_modules=["q_proj", "v_proj"],  # Q and V only - classic LoRA paper choice
)

# ── Standard: all attention projections ──────────────────────
config_standard = LoraConfig(
    r=16, lora_alpha=16, lora_dropout=0.05, bias="none", task_type="CAUSAL_LM",
    target_modules=["q_proj", "k_proj", "v_proj", "o_proj"],
)

# ── Maximum: attention + FFN (most expressive, 3x more params) ─
config_full = LoraConfig(
    r=16, lora_alpha=16, lora_dropout=0.05, bias="none", task_type="CAUSAL_LM",
    target_modules=["q_proj", "k_proj", "v_proj", "o_proj",
                   "gate_proj", "up_proj", "down_proj"],
)

# Apply adapters to model
model = get_peft_model(model, config_standard)
model.print_trainable_parameters()
# trainable params: 20,971,520 || all params: 8,051,372,032 || trainable: 0.26%
# ~21M trainable out of 8B total - this is what you train
# Optimizer states for 21M params in FP32: only ~84MB extra!

Dropout and Bias

lora_dropout: Applied to the LoRA adapter's output before adding to base. Acts as regularization. Use 0.05 for small datasets (<5K examples), 0.0 for large datasets (>10K). Too high dropout prevents learning; too low causes overfitting.

bias="none": Whether to train bias terms. Almost always leave at "none" - training biases adds minimal quality improvement but complicates merging adapters back into the base model.

What You Now Know

  • Rank r=16 is the right default for most instruction tuning tasks
  • alpha=r gives scale=1.0 (neutral); alpha=2r doubles adapter contribution
  • Start with q_proj+v_proj (classic LoRA); add k_proj+o_proj for harder tasks
  • Add gate/up/down_proj for maximum expressivity on large domain shifts
  • lora_dropout=0.05 for small datasets, 0.0 for large ones
