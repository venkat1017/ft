⏱ 60 min📊 Data Engineering🗃️ HuggingFace Datasets

The single biggest lever in fine-tuning is data quality, not model size or hyperparameters. 500 perfectly curated examples routinely beats 50,000 noisy ones. This lesson covers how to build, inspect, and format a fine-tuning dataset that produces a model that actually works.

Chat Templates: The Hidden Format

Every instruct model was trained with a specific conversation format baked in. If you fine-tune with the wrong format, the model ignores your data - it's reading the wrong "language." The format is called a chat template, and it varies by model family.

chat_templates.py - the three most important formats
CRITICAL
# ── Llama-3 format (Meta) ─────────────────────────────────────
llama3_template = """<|begin_of_text|>
<|start_header_id|>system<|end_header_id|>
You are a helpful coding assistant.<|eot_id|>
<|start_header_id|>user<|end_header_id|>
Write a binary search function.<|eot_id|>
<|start_header_id|>assistant<|end_header_id|>
def binary_search(arr, target):
    ...<|eot_id|>"""

# ── Mistral / Zephyr format ───────────────────────────────────
mistral_template = """[INST] You are a helpful assistant.
Write a binary search function. [/INST]
def binary_search(arr, target):
    ...</s>"""

# ── ChatML format (OpenHermes, many others) ───────────────────
chatml_template = """<|im_start|>system
You are a helpful coding assistant.<|im_end|>
<|im_start|>user
Write a binary search function.<|im_end|>
<|im_start|>assistant
def binary_search(arr, target):
    ...<|im_end|>"""

# ── Using tokenizer.apply_chat_template() (safest approach) ──
from transformers import AutoTokenizer

tokenizer = AutoTokenizer.from_pretrained("meta-llama/Meta-Llama-3-8B-model-Instruct")
messages = [
    {"role": "system",    "content": "You are a helpful coding assistant."},
    {"role": "user",      "content": "Write a binary search function."},
    {"role": "assistant", "content": "def binary_search(arr, target):
    ..."},
]
# Let the tokenizer handle the template - never hardcode it yourself
formatted = tokenizer.apply_chat_template(
    messages,
    tokenize=False,        # return string, not tokens
    add_generation_prompt=False,  # True during inference, False during training
)

Building Your Dataset

build_dataset.py - from raw examples to HuggingFace Dataset
CORE CODE
from datasets import Dataset
import json

# Your raw data - can come from anywhere
# Format: list of conversation dicts
raw_examples = [
    {
        "messages": [
            {"role": "user",      "content": "Explain dropout regularization."},
            {"role": "assistant", "content": "Dropout randomly zeroes..."},
        ]
    },
    # ... hundreds more ...
]

# ── Quality filters - apply BEFORE creating dataset ───────────
def is_high_quality(example):
    msg = example["messages"]

    # 1. Must have at least user + assistant turn
    if len(msg) < 2: return False

    # 2. Assistant response must be substantive
    assistant_turns = [m for m in msg if m["role"] == "assistant"]
    if any(len(m["content"]) < 50 for m in assistant_turns): return False

    # 3. No truncated responses (model cut off mid-sentence)
    last = assistant_turns[-1]["content"]
    if last.endswith(("...", "[cont")): return False

    # 4. Not too long (would get truncated during training anyway)
    total_chars = sum(len(m["content"]) for m in msg)
    if total_chars > 8000: return False

    return True

filtered = [e for e in raw_examples if is_high_quality(e)]
print(f"Kept {len(filtered)}/{len(raw_examples)} examples ({len(filtered)/len(raw_examples):.0%})")

# ── Create HuggingFace Dataset ────────────────────────────────
dataset = Dataset.from_list(filtered)

# 90/10 train/val split
split   = dataset.train_test_split(test_size=0.1, seed=42)
print(split)
# DatasetDict({train: Dataset(500 rows), test: Dataset(55 rows)})

# Save locally and push to hub
split.save_to_disk("my_dataset")
# split.push_to_hub("yourusername/my-dataset")  # optional

The Loss Masking Trick

During training, you only want the model to learn from the assistant's response - not from the user's question. Without loss masking, the model wastes gradient updates trying to predict user turns, which hurts sample efficiency and can cause the model to start generating user-style text spontaneously.

loss_masking.py - only train on assistant tokens
IMPORTANT
import torch

# After tokenizing a conversation, labels = token_ids for assistant turns
# labels = -100 for system/user tokens (PyTorch cross_entropy ignores -100)

def mask_user_tokens(input_ids, tokenizer):
    """Set labels to -100 for all non-assistant tokens."""
    labels = input_ids.clone()

    # Find where assistant turns start/end using special tokens
    # This is model-specific - here shown for Llama-3 style
    assistant_start_token = tokenizer.encode("<|start_header_id|>assistant")[0]
    eot_token             = tokenizer.encode("<|eot_id|>")[0]

    in_assistant = False
    for i, tok in enumerate(input_ids):
        if tok == assistant_start_token:
            in_assistant = True
        elif tok == eot_token and in_assistant:
            in_assistant = False
        elif not in_assistant:
            labels[i] = -100   # ignored by cross_entropy

    return labels

# TRL's SFTTrainer handles this automatically when you pass
# dataset_text_field="messages" with conversations format.
# But understanding it helps when debugging loss plateaus.
⚠️
Common mistake: Training without loss masking. The model learns to predict both the question AND the answer, wasting 50%+ of gradient signal. You'll notice this if your training loss is suspiciously low - it's fitting to easy user-turn tokens.

What You Now Know

  • Each model family has its own chat template - always use tokenizer.apply_chat_template()
  • Quality filters: min length, no truncation, reasonable total length
  • Loss masking: labels=-100 for user/system tokens, model only learns from assistant turns
  • 500 high-quality examples beats 50K noisy ones - curate ruthlessly
  • HuggingFace Datasets library for loading, splitting, and pushing to hub
